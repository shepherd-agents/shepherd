*How-to guide. New to Shepherd? Start with the tutorial. For exact APIs, see the reference.*

Guides are **task recipes**: each does one named job and keeps one shape,
job, prerequisites, steps, expected result, and what to do when it fails.
They assume you have already met Shepherd (start with the
[tutorial](../tutorials/first-shepherd-app.md) if not).

Available now, in reading order, each building on the one before; the
runnable code in each works today against the deterministic offline provider:

- [Grant a task repository access](grant-repo-access.md)
- [Deterministic demo](deterministic-demo.md)
- [Debug your first run](debug-your-first-run.md)
- [Test Shepherd code](test-shepherd-code.md)
- [Route tasks to models](route-tasks-to-models.md)
